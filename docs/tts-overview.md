# 音声合成の全体像

## 3つのアプローチ

### 1. テキスト → 音声（TTS: Text-to-Speech）
テキストから音声を生成する。一番基本的なアプローチ。

```
テキスト → 音声合成エンジン → 音声データ(WAV等)
```

- VOICEVOX, COEIROINK — あらかじめ用意されたキャラの声
- OpenAI TTS, Google Cloud TTS — クラウドAPI、多言語対応

### 2. 声 → 声（Voice Conversion / ボイスチェンジ）
自分の声を入力して、別の人の声に変換する。

```
自分の声 → 声質変換モデル → 別人の声
```

- RVC (Retrieval-based Voice Conversion) — 数分の音声サンプルで声をクローン
- リアルタイム配信向け

### 3. テキスト → 特定の人の声（TTS + Voice Cloning）
テキストから、特定の人の声で音声を生成する。1と2の合体。

```
テキスト + 声のサンプル → 音声データ（その人の声で喋る）
```

- ElevenLabs — 数秒〜数分のサンプルで声をクローン
- CoeFont — 声優が公式にライセンス提供。権利クリア
- Style-Bert-VITS2 — ローカルで動く。自分で学習データを用意して声を作れる
- GPT-4o — OpenAIのリアルタイムAPI

## TTSの軽い/重いの違い

| | 軽いTTS (VOICEVOX等) | 重いTTS (ElevenLabs, GPT-4o等) |
|---|---|---|
| 声の種類 | 固定キャラ | 任意（クローン可） |
| 感情・抑揚 | パターン固定 | 文脈から自動推定 |
| 言語 | 1言語 | 多言語 |
| モデルサイズ | 数十MB〜数百MB | 数GB〜 |
| 推論 | CPUで数秒 | GPU推奨 |

軽いTTSが軽い理由: タスクが「狭い」から。固定キャラ・固定パターン・1言語に割り切っている。

重いTTSは「文脈から感情を推定する」「任意の声を再現する」など、言語理解に近い領域に入るためモデルが大きくなる。
